      This section contains route handlers for various GET requests.
      These handlers respond to different routes, 
      providing JSON objects or processing query parameters based on the route.


      // Handle GET Requests for Different Routes

      // Handle GET request for the root URL
      app.get("/", (req, res) => {
        res.send({ data: "This is the first request handler" });
      });

      // Handle GET request for "/dog" route
      app.get("/dog", (req, res) => {
        console.log(req.params);
        res.send({ dog: "Woof" });
      });

      // Handle GET request for "/dog" route with two parameters
      app.get("/dog/:firstValue/:someOtherValue", (req, res) => {
        console.log(req.params);
        console.log(req.params.someOtherValue);
        res.send({ dog: "Meow" });
      });

      // Handle GET request for "/cat" route and display query parameters
      app.get("/cat", (req, res) => {
        console.log(req.query);
        res.send({ data: req.query });
      });

  Block Scope: 
  Variables declared with let or const are limited to the block (enclosed by {}) they are defined in. 
  They are not accessible outside the block.

  Function Scope: 
  Variables declared with var have function-level scope. 
  They are accessible within the function in which they are defined.

  const:
  const is used to declare variables that should not be reassigned. It is not immutable, 
  meaning you can change the internal properties of a const object.
  const variables are stored in memory, just like other variables.
  var is a globally scoped variable. It is accessible from anywhere in the code.

  let:
  let is block-scoped like const, but it can be initialized without a value and can be reassigned.
  Unlike const, you can redeclare a variable using let in the same scope.

  Hoisting:
  Hoisting refers to the behavior in JavaScript where variable and function declarations are moved to the top of their containing 
  function or global scope during compilation.
  This allows you to use a function or variable before it's declared in your code.

  Callback Function:
  A callback function is a function that is passed as an argument to another function and is executed after the completion of that function.
  It is often used to handle asynchronous operations in JavaScript, such as in event handling or making API calls.
